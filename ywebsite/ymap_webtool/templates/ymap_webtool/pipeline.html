STEP 1: Mutation at DNA level to protein level (optional)

When the input file contain mutations at DNA level, it is translated to protein level for yMap to be able to process. All the non-synonymous mutation are reported in a file that will be use as input by the core function of yMap.

STEP 2: Mapping the mutations to protein features and post translational modifications.

The mutations at protein level are mapped to protein features and post translational modifications. yMap generates a report file for each types of protein features and post translational modifications that contains/overlaps with mutations from the input file.

STEP 3: GO enrichment analysis

Gene Ontology (GO) enrichment effectively predicts which biological process, cellular component and molecular functions are potentially disturbed by a subset of the mutations. It identifies GO terms that are overrepresented in sets of the genes to which mutations have been mapped, as compared to the background of the complete yeast genome. The significance of enriched terms is computed using the Orange Bioinformatics package.

STEP 4: Biogrid id

The list of BioGrid IDs for each genes to which mutations have been mapped is generated.
